# NBA预测 H5 需求文档 (PRD)

## 一、 页面 UI 及交互说明

### 1. 核心信息展示
* **页面标题**：NBA预测
* **活动标语**：玩竞猜，送吧币
* **竞猜截止时间**：显示格式如“1月8日 07:00”
* **对阵信息**：
 * 显示主客队Icon及队伍名称，中间显示“VS”。
* **盘口选项**：
 * 显示竞彩篮球-大小分的盘口数和大小选项（如：总分高于200.5分 / 总分低于200.5分）。
* **提交按钮**：
 * 用户未选择大或者小则提交按钮置灰，无法点击。

### 2. 专家与AI分析模块
* **专家预测**：
 * 包含专家头像、专家昵称、战绩标签（如“近10中8”等）。
 * 展示专家详细的分析文案。
 * *示例分析文案：“我偏好‘大于200.5分’。这场比赛很可能会演变为一场对攻大战。进攻方面，两队都拥有能够撕开防线的得分点，可以持续压迫防守阵型并创造出手回合。防守方面，双方都缺乏顶级的防守资源，尤其在轮转换位和护框环节存在明显短板，很难限制对方在禁区和外线的双重打击。考虑到比赛的重要程度或双方近期的开放态度，高强度防守的可能性较低，比赛节奏大概率会被不断提速，进球数应该不会少。”*
* **AI 观点**：
 * 包含AI头像（专家助理）。
 * 展示基于专家分析提炼的AI观点文案。
 * *示例AI观点文案：“结合专家分析不难得出，两边的防守都不太行，护框和轮换都有明显漏洞，想按住对方得分有点难。而且两队都有着撕裂防线的点，进攻火力能一直续上。这比赛节奏只会被打超快，要演变成对攻战。”*
* **注**：专家预测和AI观点的展示内容取决于用户所在的分组（详见下方 AB 实验分组表）。

### 3. 弹窗交互说明
* **意愿询问弹窗**：
 * 触发条件：用户选择选项并点击“提交”按钮后弹出。
 * 文案：“您是否愿意在后续的竞猜活动中继续收到（专家名称/AI名称）的专家分析？”
 * *注：括号内的名称需根据用户所在的实验分组动态展示（展示对应的「专家昵称」字段，如：磊哥看球、娜姐看球、大数据智能预测等）。*
 * 按钮：[愿意] / [不愿意]
* **提交成功提示**：
 * 触发条件：用户点击意愿弹窗的任一按钮后弹出。
 * 文案：“提交成功。比赛结束后我们将在竞猜正确中进行抽奖，如中奖我们将通过短信和站内信进行告知。”
 * 按钮：[知道了]

### 4. 活动规则
1. **活动时间**：待定
2. **参与方式**：用户每次活动可参与一次竞猜，竞猜成功后可获得抽奖机会。
3. **领奖须知**：如您已获奖，我们会将获奖信息以短信形式和站内信形式发送给您，请您在开奖后7个工作日内，再次进入活动页面填写收货信息，奖品将于活动结束后30个工作日内发货，逾期填写将视为自动放弃领奖资格，如有疑问可联系客服
4. **奖品信息**：本次活动所展示的奖品信息仅供参考，最终发放的奖品请以实物为准
5. **注意事项**：Apple未以任何形式参与和赞助本活动
6. **免责声明**：
 * （1）平台仅提供竞猜技术服务，不对赛事结果、竞猜胜负承担任何责任，用户需自行判断并承担竞猜风险。
 * （2）因不可抗力（如地震、火灾、网络故障、政策调整）导致平台无法正常运营或用户无法参与竞猜，平台不承担赔偿责任，但会尽力协助用户减少损失。
 * （3）本规则最终解释权归平台所有，平台有权根据国家法律法规、市场情况及运营需求对规则进行修订，修订后将通过平台公告通知用户，修订内容自公告发布之日起生效。

## 二、 业务功能规则表

| 模块 | 序号 | 功能点 | 规则说明 |
| :--- | :--- | :--- | :--- |
| **全局** | 1.1 | 入口 | 无需登录即可进入 H5 页面预览；参与活动（选择专家）时需登录，未登录用户点击选择按钮后跳转登录页。<br>用户通过文字直播进入此页面，编辑通过后台发送h5链接。 |
| **全局** | 1.2 | 频控规则 | 用户每次活动仅限提交一次预测，提交后选项锁定且提交按钮置灰显示“已提交”。 |
| **对阵** | 2.1 | 主客队 | 显示主客队的队名和球队icon。 |
| **对阵** | 2.2 | 盘口 | 显示竞彩篮球-大小分的盘口数和大小选项。 |
| **交互** | 3.1 | 提交 | 用户未选择大或者小则提交按钮置灰，无法点击。 |
| **交互** | 3.2 | 提交后弹窗 | 用户点击提交后，先弹窗询问是否后续竞猜活动中继续收到专家分析（文案需根据用户分组动态展示对应的专家名称或AI名称），用户点击愿意或不愿意后弹窗提示提交成功。 |
| **后台** | 4.1 | 数据记录 | 记录每个用户每次预测的选项、是否命中、是否愿意继续接受专家分析。 |
| **策略** | 5.1 | 专家分析 | 用户进入此页面后会均匀分组，每个组显示不同的专家分析内容。专家的头像、昵称、战绩标签（准确度）、专家建议均按照表格内的进行分配展示。 |
| **策略** | 5.2 | AI观点 | AI观点是否展示取决于用户的分组，根据分组显示头像、昵称、性别、选项。 |
| **运营** | 6.1 | 场次 | 暂定测试10场比赛，每场比赛均发放1000吧币，这1000吧币均分给参与预测的20%用户，随机抽取。 |
| **运营** | 6.2 | 链接 | 前期先测试一场比赛，后续会继续添加。 |
